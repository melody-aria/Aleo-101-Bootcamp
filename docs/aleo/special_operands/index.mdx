# Aleo Special Operands

以下列表显示了 Aleo 指令支持的特殊操作数。

## 特殊操作数表

名称| 描述
---|---
[block.height](#blockheight)| 返回 finalize 作用域内区块的高度
[block.timestamp](#blocktimestamp)| 返回 finalize 作用域内当前区块的 unix 时间戳
[self.signer](#selfsigner)| 返回发起转换的用户地址
[self.caller](#selfcaller)| 返回程序的直接调用者地址
[network.id](#networkid)| 返回程序执行所在网络的 ID
[edition](#edition)| 返回程序的版本号 (u16)
[checksum](#checksum)| 返回程序字符串的 SHA3-256 哈希
[program_owner](#programowner)| 返回提交部署交易的账户地址

## 群生成元表

名称| 描述
---|---
[group::GEN](#groupgen)| 椭圆曲线的素阶子群生成元
[aleo::GENERATOR](#aleogenerator)| 用于 Aleo 账户推导的点 G
[aleo::GENERATOR_POWERS](#aleogeneratorpowers)| Aleo 签名和加密方案的完整群基

## 规范

以下是 Aleo 虚拟机 (AVM) 中各特殊操作数的规范。

### `network.id`

[返回顶部](#table-of-special-operands)

#### 描述

返回程序执行所在网络的 ID。这对于管理特定网络的程序行为非常有用。 `network.id` 命令必须在 finalize 块中调用。

目前支持的网络 ID 如下：

  * 0: Mainnet
  * 1: Testnet
  * 2: Canarynet


#### 示例用法


    assert.eq network.id 0u64;  // 适用于主网

### `block.height`

[返回顶部](#table-of-special-operands)

#### 描述

返回 finalize 作用域内区块的高度。
`block.height` 命令必须在 finalize 块中调用。 返回类型为 `u64`。

#### 示例用法


    assert.eq block.height 100u64;

### `block.timestamp`

[返回顶部](#table-of-special-operands)

#### 描述

返回 finalize 作用域内当前区块的 unix 时间戳。
`block.timestamp` 命令必须在 finalize 块中调用。
返回类型为 `i64`。

#### 示例用法


    assert.eq block.timestamp 1767976339i64;

### `self.signer`

[返回顶部](#table-of-special-operands)

#### 描述

返回发起转换的用户地址。

#### 示例用法


    assert.eq self.signer aleo1...;

### `self.caller`

[返回顶部](#table-of-special-operands)

#### 描述

返回程序的直接调用者地址。

#### 示例用法


    assert.eq self.caller aleo1...;

### `edition`

[返回顶部](#table-of-special-operands)

#### 描述

返回程序的版本号，作为无符号 16 位整数 (`u16`)。`edition` 在初始部署时必须为 `0u16`。对于每次有效的升级，它必须以 1 递增。

此操作数仅在 `finalize` 作用域内可用，用于程序的可升级性。

#### 示例用法


    assert.eq edition 0u16;  // 检查是否为初始部署


::: note[注意]
您也可以通过限定程序名称来引用其他程序的元数据，例如 `Program::edition(credits.aleo)`、`Program::edition(foo.aleo)`。您需要在 Leo 文件中导入该程序才能使用此语法。
:::

### `checksum`

[返回顶部](#table-of-special-operands)

#### 描述

返回一个 32 字节数组 (`[u8; 32u32]`)，表示程序字符串的 SHA3-256 哈希。它是程序代码的唯一指纹。

在任何可升级程序的部署中都需要 `checksum`，用于验证部署的代码是否符合预期。

此操作数仅在 `finalize` 作用域内可用。

#### 示例用法


    assert.eq checksum <EXPECTED_CHECKSUM>;  // 验证程序代码是否与预期哈希匹配


::: note[注意]
您也可以通过限定程序名称来引用其他程序的元数据，例如 `Program::checksum(credits.aleo)`、`Program::checksum(foo.aleo)`。您需要在 Leo 文件中导入该程序才能使用此语法。
:::

### `program_owner`

[返回顶部](#table-of-special-operands)

#### 描述

返回提交部署交易的账户的 `地址`。

在任何可升级程序的部署中都需要 `program_owner`，可用于强制执行程序升级的访问控制。

此操作数仅在 `finalize` 作用域内可用。

#### 示例用法


    assert.eq program_owner <ADMIN_ADDRESS>;  // 将升级限制为特定管理员


::: note[注意]
您也可以通过限定程序名称来引用其他程序的元数据，例如 `Program::program_owner(credits.aleo)`、`Program::program_owner(foo.aleo)`。您需要在 Leo 文件中导入该程序才能使用此语法。
:::


::: warning[警告]
在可升级性引入之前部署的程序没有 `program_owner`。尝试访问它会导致运行时错误。
:::

### `group::GEN`

[返回顶部](#table-of-group-generators)

#### 描述

椭圆曲线的素阶子群生成元。 它用于 Twisted Edwards 曲线上的通用椭圆曲线运算，并在特定转换操作中作为默认值之一。

#### 示例用法


    add group::GEN group::GEN into r0;
    mul group::GEN 2scalar into r1;

### `aleo::GENERATOR`

[返回顶部](#table-of-group-generators)

#### 描述

用于 Aleo 账户推导的点 `G`，其中 `view_key * G = address`。用于 `g_scalar_multiply` 进行 Aleo 账户和签名推导。它是通过在域字符串 `"AleoAccountEncryptionAndSignatureScheme0"` 上调用 `hash_to_curve` 生成的群基的第一个元素。这些群基专门用于 Aleo 账户和签名。

#### 示例用法


    function derive_address:
        input r0 as scalar.public;  // view_key
        input r1 as address.public; // 期望的地址
        // 从 view_key 推导地址
        mul aleo::GENERATOR r0 into r2;
        cast r2 into r3 as address;
        assert.eq r1 r3;


::: note[注意]
请参阅 [账户文档](../../../concepts/fundamentals/accounts/index.mdx#generate-an-address) 了解有关该生成元在账户推导中如何使用的更多详细信息。
:::

### `aleo::GENERATOR_POWERS`

[返回顶部](#table-of-group-generators)

#### 描述

Aleo 签名和加密方案的群基，通过在域字符串 `"AleoAccountEncryptionAndSignatureScheme0"` 上调用 `hash_to_curve` 生成。专门用于 Aleo 账户和签名。

数组长度为 253，有效索引为 `0u32` 到 `252u32`。

#### 示例用法


    function example:
        input r0 as scalar.public;
        // 访问第一个生成元幂（等同于 aleo::GENERATOR）
        mul aleo::GENERATOR_POWERS[0u32] r0 into r1;

        // 验证与 aleo::GENERATOR 的等价性
        mul aleo::GENERATOR r0 into r2;
        assert.eq r1 r2;

        // 这两者是等价的
        assert.eq aleo::GENERATOR aleo::GENERATOR_POWERS[0u32];
