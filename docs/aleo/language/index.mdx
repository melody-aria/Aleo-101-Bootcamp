# Aleo 指令语言指南

### 静态类型

Aleo 指令是一门**静态类型语言** ，这意味着在执行电路之前，我们必须知道每个变量的类型。

### 必须显式声明类型

Aleo 指令中没有 `undefined` 或 `null` 值。在定义新变量时，**必须显式声明值的类型** 。

### 值传递

Aleo 指令中的表达式始终是**值传递** 的，这意味着当它们被用作函数输入或在赋值语句的右侧时，其值总是被复制。

### 基于寄存器

Aleo 指令中没有变量名。 所有变量都存储在寄存器中，表示为 `rX`，其中 `X` 是从 0 开始的非负整数：`r0, r1, r2` 等等。

## 数据类型与值

### 布尔值

Aleo 指令支持传统的 `true` 或 `false` 布尔值。在语句中使用布尔值时，必须显式指定 `boolean` 类型。


    function main:
        input r0: boolean.private;

### 整数

Aleo 指令支持有符号整数类型 `i8`、`i16`、`i32`、`i64`、`i128` 以及无符号整数类型 `u8`、`u16`、`u32`、`u64`、`u128`。


    function main:
        input r0: u8.public;


::: info[信息]
更高位宽的整数在电路中会产生更多的约束，从而可能导致计算速度变慢。
:::

### 域元素

Aleo 指令支持 `field` 类型，用于表示椭圆曲线基域中的元素。 这些是无符号整数，小于基域的模数，因此最大的域元素为 `8444461749428370424248824938781546531375899335154063827935233455917409239040field`。


    function main:
        input r0: field.private;

### 群元素

传入 Aleo 指令编译器的椭圆曲线上的仿射点集构成了一个群。 该曲线是一个 Twisted Edwards 曲线，其中 `a = -1`，`d = 3021`。 Aleo 指令支持该群的一个子群（由一个生成元生成）作为原始数据类型。 群元素由其点的 x 坐标表示；例如， `2group` 表示点 `(2, 5553594316923449299484601589326170487897520766531075014687114064346375156608)`。 生成元点为 `1540945439182663264862696551825005342995406165131907382295858612069623286213group`。


    function main:
        input r0: group.private;

#### 群生成元

Aleo 提供了多个内置的群生成元，可在程序中使用：

名称| 描述
---|---
`group::GEN`| 椭圆曲线素数阶子群的生成元。
`aleo::GENERATOR`| 单个群点（不可索引）。用于 Aleo 账户派生的点 `G`，满足 `view_key * G = address`。
`aleo::GENERATOR_POWERS`| 用于 Aleo 签名和加密方案的群基数组。使用 `aleo::GENERATOR_POWERS[Nu32]` 进行索引。

**`group::GEN`** 是椭圆曲线素数阶子群的生成元，用于通用的椭圆曲线运算。

**`aleo::GENERATOR`** 是满足 `view_key * G = address` 的点 `G`，用于 `g_scalar_multiply` 进行 Aleo 账户和签名派生。它是 `aleo::GENERATOR_POWERS` 的第一个元素。

**`aleo::GENERATOR_POWERS`** 是 Aleo 签名和加密方案的群基，通过对域字符串 `"AleoAccountEncryptionAndSignatureScheme0"` 调用 `hash_to_curve` 生成。该数组专门用于 Aleo 账户和签名。数组长度为 253，有效索引范围为 `0u32` 到 `252u32`。


    function example:
        input r0 as scalar.public;
        input r1 as address.public;
        // 使用椭圆曲线生成元
        add group::GEN group::GEN into r2;

        // 使用 Aleo 账户生成元从视图密钥派生地址
        mul aleo::GENERATOR r0 into r3;

        // 访问 GENERATOR_POWERS 的第一个元素（与 aleo::GENERATOR 相同）
        mul aleo::GENERATOR_POWERS[0u32] r0 into r4;

        // 这两个操作产生相同的结果
        assert.eq r3 r4;

        // Cast the group element to an address
        cast r3 into r5 as address;
        assert.eq r1 r5;

### 标量元素

Aleo 指令支持 `scalar` 类型，用于表示椭圆曲线子群定义的标量域中的元素。 这些是无符号整数，小于标量域的模数，因此最大的标量为 `2111115437357092606062206234695386632838870926408408195193685246394721360382scalar`。


    function main:
        input r0: scalar.private;

### 地址

地址类型的定义使得编译器可以对地址的解析和操作进行优化。


    function main:
        input r0: address.private;

### 签名

Aleo 使用 Schnorr 签名方案，通过 Aleo 私钥对消息进行签名。 签名可以在 Aleo 指令中使用 [`sign.verify`](../opcodes/index.mdx#signverify) 指令进行验证。


    sign.verify sign069ju4e8s66unu25celqycvsv3k9chdyz4n4sy62tx6wxj0u25vqp58hgu9hwyqc63qzxvjwesf2wz0krcvvw9kd9x0rsk4lwqn2acqhp9v0pdkhx6gvkanuuwratqmxa3du7l43c05253hhed9eg6ppzzfnjt06fpzp6msekdjxd36smjltndmxjndvv9x2uecsgngcwsc2qkns4afd r1 r2 into r3;

### 动态记录

`record.dynamic` 是任意记录的固定大小通用表示，可跨 [动态调用](#dynamic-dispatch) 边界使用。与静态记录不同，其大小是恒定的，无论其数据字段的数量或类型如何。

字段| 类型| 描述
---|---|---
`owner`| `address`| 记录的所有者
`_root`| `field`| 记录数据条目的 Merkle 根
`_nonce`| `group`| 记录的 nonce
`_version`| `u8`| 记录版本

`record.dynamic` 无法直接实例化。它必须作为输入接收、从 [`call.dynamic`](#calldynamic) 返回，或通过转换静态记录来创建：


    cast r0 into r1 as record.dynamic;

`owner` 字段可以直接访问（`r1.owner`）。其他字段需要使用 [`get.record.dynamic`](#getrecorddynamic)。


::: note[注意]
接受 `record.dynamic` 的函数**不会** 验证所有权或废除该记录。只有接收静态记录输入的函数才会消耗（废除）该记录。
:::

### 动态 Future

`future.dynamic` 仅由 [`call.dynamic`](#calldynamic) 在被调用方返回 future 时生成。与静态 future 一样，它必须由 `async` 消费并在 finalize 中等待：


    finalize my_function:
        input r0 as future.dynamic;
        await r0;

`future.dynamic` 不能由函数直接输出——它必须通过 `async` 指令包装在静态 future 中。

## Aleo 程序布局

一个 Aleo 程序包含 [程序 ID](#programid)、[导入](#import)、[函数](#function)、[闭包](#closure)、[结构体](#struct)、[记录](#record)、 [映射](#mapping) 和 [Finalize](#finalize) 的声明。只有导入必须位于文件顶部，顺序有强制要求。 声明在程序文件中是局部可访问的。 如果需要使用另一个程序文件中的声明，则必须通过导入来获取。

### 程序 ID

程序 ID 声明格式为 `{name}.{network}`。 `name` 的第一个字符必须小写。 `name` 可以包含小写字母、数字和下划线。 目前，`aleo` 是唯一支持的 `network` 域。


    program hello.aleo; // 有效

    program Foo.aleo;   // 无效
    program baR.aleo;   // 无效
    program 0foo.aleo;  // 无效
    program 0_foo.aleo; // 无效
    program _foo.aleo;  // 无效

### 导入

导入声明格式为 `import {ProgramID};`。
导入通过其他声明的程序 ID 获取它们，并将它们引入当前文件作用域。 你可以导入下载到 `imports` 目录中的依赖项。


    import foo.aleo; // 将 `foo.aleo` 程序导入 `hello.aleo` 程序。

    program hello.aleo;

### 函数

函数声明格式为 `function {name}:`。
函数包含可以计算值的指令。 函数必须位于程序的当前作用域内才能被调用。


    function foo:
        input r0 as field.public;
        input r1 as field.private;
        add r0 r1 into r2;
        output r2 as field.private;

#### 函数输入

函数输入声明格式为 `input {register} as {type}.{visibility};`。
函数输入必须在函数名称声明之后立即声明。


    // 函数 `foo` 接受单个输入 `r0`，类型为 `field`，可见性为 `public`。
    function foo:
        input r0 as field.public;

#### 函数输出

函数输出声明格式为 `output {register} as {type}.{visibility};`。
函数输出必须在函数定义末尾声明。


    ...
        output r0 as field.public;

#### 调用函数

在 Aleo 协议中，调用函数会创建一个可以消费和生成链上记录的 transition（状态转换）。 使用 `aleo run` CLI 命令向函数传递输入并执行程序。
在测试网中，程序函数不能调用同一程序内的其他函数。 如果你想开发在程序内部调用的"辅助函数"，可以尝试编写 `闭包`。

#### 调用导入的函数

Aleo 程序可以使用 `call {program}/{function} {register} into {register}` 指令从外部调用其他 Aleo 程序。


    import foo.aleo;

    program bar.aleo;

    function call_external:
        input r0 as u64.private;
        call foo.aleo/baz r0 into r1; // 外部调用 foo.aleo 中的函数 `baz`，传入参数 `r0` 并将结果存入 `r1`。
        output r1 as u64.private;

### 闭包

闭包声明格式为 `closure {name}:`。
闭包包含可以计算值的指令。 闭包是不可直接执行的辅助函数。 闭包可以被其他局部或外部函数调用，但不能返回记录，也不能执行调用或等待。


    closure foo:
        input r0 as field;
        input r1 as field;
        add r0 r1 into r2;
        output r2 as field;

#### 调用闭包

Aleo 程序可以使用 `call {name} {register} into {register}` 指令从内部调用其他 Aleo 闭包。


    program bar.aleo;

    function call_internal:
        input r0 as u64.private;
        call foo r0 into r1; // 内部调用闭包 `foo`，传入参数 `r0` 并将结果存入 `r1`。
        output r1 as u64.private;

### 结构体

结构体是一种数据类型，声明格式为 `struct {name}:`。
结构体包含组件声明 `{name} as {type}`。


    struct array3:
        a0 as u32;
        a1 as u32;
        a2 as u32;

在程序中实例化 `struct` 使用 `cast` 指令。


    function new_array3:
        input r0 as u32.private;
        input r1 as u32.private;
        input r2 as u32.private;
        cast r0 r1 r2 into r3 as array3;
        output r3 as array3.private;

### 外部结构体

外部结构体允许你使用在其他已导入的 Aleo 程序中定义的数据结构。

  * **命名空间语法** ：外部结构体使用斜杠运算符引用：`program_name.aleo/struct_name`。
  * **使用方式** ：一旦程序被导入，其结构体可用作变量类型、函数输入和返回类型。
  * **类型转换** ：与记录不同，外部结构体可以使用 `cast` 指令在本地实例化，因为它们是数据模式，而非状态承载对象。




    import token_registry.aleo;

    transition get_metadata:
        input r0 as field.public;
        // 如果你有所需的字段，可以转换外部结构体
        cast r0 0u64 18u8 into r1 as token_registry.aleo/TokenInfo;
        output r1 as token_registry.aleo/TokenInfo.private;

### 数组

数组字面量写作 `[{value}, {value}, ..]`，其中所有值具有相同的类型。例如，


    [true, false, true]

数组的类型包含元素类型和数组长度 `[{type}; {length}]`。此示例的类型为


    [boolean; 3u32]

可以使用 `cast` 操作码初始化数组。


    function new_array:
        input r0 as boolean.private;
        input r1 as boolean.private;
        input r2 as boolean.private;
        cast r0 r1 r2 into r3 as [boolean; 3u32];
        output r3 as [boolean; 3u32].private;

可以使用 `{name}[{index}]` 索引数组。


    function get_array_element:
        input r0 as [boolean; 4u32].public;
        input r1 as u32.public;
        r0[r1] into r2;
        output r2 as boolean.public;

数组可以嵌套。


    [[true, false, true, false], [false, true, false, true]]


    function get_nested_array_element:
        input r0 as [[boolean; 4u32]; 2u32].public;
        r0[0u32][1u32] into r1;
        output r1 as boolean.public;


::: info[信息]
Aleo 指令目前仅支持固定长度的静态数组。最大数组大小为 **2,048 个元素** （在 Aleo Stack v4.6.0 中从 512 增加，以支持验证 [`snark.verify`](../opcodes/index.mdx#snarkverify)).
:::

### 记录

[记录](../../../concepts/fundamentals/records/index.mdx) 类型声明格式为 `record {name}:`。
记录包含组件声明 `{name} as {type}.{visibility};`。
记录数据结构必须包含 `owner` 声明，如下所示。
当将记录作为输入传递给程序函数时，还必须包含 `_nonce as group.{visibility}` 声明。


    record token:
        // 代币所有者。
        owner as address.private;
        // 代币数量。
        amount as u64.private;

在程序中实例化 `record` 使用 `cast` 指令。


    function new_token:
        input r0 as address.private;
        input r1 as u64.private;
        input r2 as u64.private;
        cast r0 r1 r2 into r3 as token.record;
        output r3 as token.record;

### 外部记录

你可以引用在其他程序中定义的记录，来指定你的程序期望接收或传递的数据类型。

  * **命名空间语法** ：外部记录使用相同的斜杠运算符：`program_name.aleo/record_name`。
  * **只读访问** ：你可以访问传入函数的外部记录的字段。
  * **所有权规则** ：程序不能创建（`cast`）或花费（生成 nullifier）属于其他程序的记录。只有定义该记录的程序才有权更改其状态。

操作| 内部记录| 外部记录
---|---|---
作为输入传递| 允许| 允许
访问字段| 允许| 允许
通过 `cast` 实例化| 允许| 禁止
花费/销毁| 允许| 禁止

**传递模式：**

由于无法花费外部记录，标准模式是将其作为输入接收，并将其传递给定义程序内的外部调用。


    import credits.aleo;

    // 你的程序充当中间人
    function transfer_on_behalf:
        input r0 as credits.aleo/credits.record;
        input r1 as address.private;
        input r2 as u64.private;
        // We cannot 'spend' r0 here.
        // We must call credits.aleo to handle the logic.
        call credits.aleo/transfer_private r0 r1 r2 into r3 r4;
        output r3 as credits.aleo/credits.record;
        output r4 as credits.aleo/credits.record;

### 特殊操作数

#### self.signer

`self.signer` 操作数返回发起该转换的用户地址。
这在需要修改原始调用方而非自身状态的中间程序中特别有用。

在下面的示例中，`transfer_public_as_signer` 函数使用 `self.signer` 从原始用户的账户中扣减余额，而非从中间程序的账户扣减。


    // `transfer_public_as_signer` 函数将指定金额
    // 从签名者的 `account` 发送到接收者的 `account`。
    function transfer_public_as_signer:
        // 输入接收者。
        input r0 as address.public;
        // 输入金额。
        input r1 as u64.public;
        // Transfer the credits publicly.
        async transfer_public_as_signer self.signer r0 r1 into r2;
        // Output the finalize future.
        output r2 as credits.aleo/transfer_public_as_signer.future;

    finalize transfer_public_as_signer:
        // Input the signer.
        input r0 as address.public;
        // 输入接收者。
        input r1 as address.public;
        // 输入金额。
        input r2 as u64.public;
        // Decrements `account[r0]` by `r2`.
        // If `account[r0] - r2` underflows, `transfer_public_as_signer` is reverted.
        get account[r0] into r3;
        sub r3 r2 into r4;
        set r4 into account[r0];
        // Increments `account[r1]` by `r2`.
        // If `account[r1]` does not exist, 0u64 is used.
        // If `account[r1] + r2` overflows, `transfer_public_as_signer` is reverted.
        get.or_use account[r1] 0u64 into r5;
        add r5 r2 into r6;
        set r6 into account[r1];

#### self.caller

`self.caller` 操作数返回程序的直接调用方的地址。

### 映射

映射声明格式为 `mapping {name}:`。映射包含键值对，必须在程序中定义。
映射公开存储在链上，无法在映射中隐私地存储数据。


    // 链上存储 `account` 映射，以 `owner` 为键，
    // 以 `amount` 为值。
    mapping account:
        // 代币所有者。
        key as address.public;
        // 代币数量。
        value as u64.public;

#### 检查是否存在

一个检查键是否存在于映射中的命令，例如 `contains accounts[r0] into r1;`。

#### 获取

一个从映射中检索值的命令，例如 `get accounts[r0] into r1;`。

#### 获取或使用默认值

一个获取命令，在失败时使用提供的默认值，例如 `get.or_use account[r1] 0u64 into r5;`。


    // `transfer_public` 函数将指定金额
    // 从调用者的 `account` 发送到接收者的 `account`。
    function transfer_public:
        // 输入接收者。
        input r0 as address.public;
        // 输入金额。
        input r1 as u64.public;
        // Transfer the credits publicly.
        async transfer_public self.caller r0 r1 into r2;
        // Output the finalize future.
        output r2 as credits.aleo/transfer_public.future;

    finalize transfer_public:
        // Input the caller.
        input r0 as address.public;
        // 输入接收者。
        input r1 as address.public;
        // 输入金额。
        input r2 as u64.public;
        // Decrements `account[r0]` by `r2`.
        // If `account[r0] - r2` underflows, `transfer_public` is reverted.
        get account[r0] into r3;
        sub r3 r2 into r4;
        set r4 into account[r0];
        // Increments `account[r1]` by `r2`.
        // If `account[r1]` does not exist, 0u64 is used.
        // If `account[r1] + r2` overflows, `transfer_public` is reverted.
        get.or_use account[r1] 0u64 into r5;
        add r5 r2 into r6;
        set r6 into account[r1];

#### 设置

一个在映射中设置值的命令，例如 `set r0 into accounts[r0];`。

#### 移除

一个从映射中移除键值对的命令，例如 `remove accounts[r0];`。

#### 读取外部程序的映射值

程序也可以读取外部程序的映射值。这使你的程序能够访问和利用由其他程序维护的数据，从而实现可组合性。例如：


    // 导入你要读取其映射的外部程序。
    import credits.aleo;
    program another_program.aleo;

    function main:
        async main into r0;
        output r0 as another_program.aleo/main.future;

    finalize main:
        // Read value from 'account' mapping in the external 'credits.aleo' program for the specified key.
        get credits.aleo/account[aleo1...] into r0;
        // Read value from 'account' mapping in the external 'credits.aleo' program for the specified key,
        // or use the default value 0u64 if the mapping does not exist.
        get.or_use credits.aleo/account[aleo1...] 0u64 into r1;

### Finalize

Finalize 声明格式为 `finalize {name}:`。
Finalize 必须紧跟在 [函数](#function) 之后，并且必须具有相同的名称；


    // `transfer_public_to_private` 函数将指定金额
    // 从映射 `account` 转换为指定接收者的记录。
    //
    // 此函数公开揭示了发送者、接收者和指定金额。
    // 但是，使用接收者记录的后续方法可以保护接收者的隐私。
    function transfer_public_to_private:
        // 输入接收者。
        input r0 as address.private;
        // 输入金额。
        input r1 as u64.public;
        // Construct a record for the receiver.
        cast r0 r1 into r2 as credits.record;
        // Decrement the balance of the sender publicly.
        async transfer_public_to_private self.caller r1 into r3;
        // Output the record of the receiver.
        output r2 as credits.record;
        // Output the finalize future.
        output r3 as credits.aleo/transfer_public_to_private.future;

    finalize transfer_public_to_private:
        // Input the sender.
        input r0 as address.public;
        // 输入金额。
        input r1 as u64.public;
        // Retrieve the balance of the sender.
        get account[r0] into r2;
        // Decrements `account[r0]` by `r1`.
        // If `r2 - r1` underflows, `transfer_public_to_private` is reverted.
        sub r2 r1 into r3;
        // Updates the balance of the sender.
        set r3 into account[r0];


::: note[注意]
`finalize` 函数在关联函数执行的零知识证明被验证后在链上执行。 如果 finalize 函数成功，程序逻辑即被执行。
如果 finalize 函数失败，程序逻辑将被回滚。
:::

### Future

Future 等价于链上执行的调用图，并在完成执行时显式使用。 转换/电路不再从代码中隐式构建调用图，而是显式输出一个 future， 指定要在链上运行哪些代码块以及如何运行它们。

#### Future 类型

用户可以通过指定一个 `Locator` 后跟 `.future` 标签来声明 future 类型。 例如，`credits.aleo/mint_public.future`。 `function` 只能输出 future，而 finalize 块只能接收 future 作为输入。 `closure` 不能输出 future，也不能接收 future 作为输入。

#### 异步调用

用户可以通过 `async` 关键字对 finalize 块进行异步调用。 例如，`async mint_public r0 r1 into r2;`。 请注意，必须指定关联的函数。 此操作会生成一个 `Future` 作为输出。 `async` 取代了 `finalize` 命令，后者允许在输出语句之后的函数体中使用。

#### await 命令

用户可以在 finalize 块中使用 `await` 命令来计算 future。 例如，`await r0;`。 `await` 命令只能在 finalize 块中使用。 操作数必须是包含 Future 的寄存器。

#### 对 future 进行索引

包含 future 的寄存器可以使用现有的索引语法进行索引。 例如，`r0[0u32]`。 这将获取该特定索引处的 future 输入。 访问可以嵌套以匹配 future 的嵌套结构。

#### Future 示例


    program basic_math.aleo;

    mapping uses:
        key user as address.public;
        value count as i64.public;

    function add_and_count:
        input r0 as i64.private;
        input r1 as i64.private;
        add r0 r1 into r2;
        async add_and_count self.caller into r3;
        output r2 as i64.private;
        output r3 as basic_math.aleo/add_and_count.future;

    finalize add_and_count:
        input r0 as address.public;
        get.or_use uses[r0] 0i64 into r1;
        add r1 1i64 into r2;
        set r2 into uses[r0];

    function sub_and_count:
        input r0 as i64.private;
        input r1 as i64.private;
        sub r0 r1 into r2;
        async sub_and_count self.caller into r3;
        output r2 as i64.private;
        output r3 as basic_math.aleo/sub_and_count.future;

    finalize sub_and_count:
        input r0 as address.public;
        get.or_use uses[r0] 0i64 into r1;
        add r1 1i64 into r2;
        set r2 into uses[r0];

    /////////////////////////////////////////////////

    import basic_math.aleo;

    program count_usages.aleo;

    function add_and_subtract:
        input r0 as i64.private;
        input r1 as i64.private;
        call basic_math.aleo/add_and_count r0 r1 into r2 r3;
        call basic_math.aleo/sub_and_count r2 r1 into r4 r5;
        assert.eq r0 r4;
        assert.eq r3[0u32] r5[0u32];
        async add_and_subtract r3 r5 into r6;
        output r0 as i64.private;
        output r6 as count_usages.aleo/add_and_subtract.future;

    finalize add_and_subtract:
        input r0 as basic_math.aleo/add_and_count.future;
        input r1 as basic_math.aleo/sub_and_count.future;
        await r0;
        assert.eq r0[0u32] r1[0u32];
        await r1;

使用这些组件需要遵守一些规则：

  1. 如果函数有 finalize 块，则必须有且仅有一个 async 指令。
  2. 如果函数有 finalize 块，其最后一个输出必须是 future。
  3. 如果函数没有 finalize 块，则不能有 async 指令。
  4. 所有由调用创建的 future 需要按照产生的顺序作为 async 指令的输入。
  5. async 调用必须引用同一个函数。
  6. 所有调用必须在调用 async 之前完成。
  7. finalize 块中的输入 future 类型必须与它们在函数中创建的顺序匹配。
  8. finalize 中的所有 future 都必须被 await，且顺序与指定的顺序一致。
  9. 指令可以在 call、async 和 await 的调用之间交错执行。


### Finalize 命令

Aleo 指令支持以下命令以提供额外的程序功能。

#### block.height

`block.height` 命令返回程序执行所在区块的高度（最新区块高度 + 1）。
这对于管理程序基于时间的访问控制非常有用。
`block.height` 命令必须在 finalize 块中调用。


    assert.eq block.height 100u64;

#### block.timestamp

`block.timestamp` 命令在 finalize 作用域内返回当前区块的 Unix 时间戳。
这对于管理程序基于时间的访问控制非常有用。
`block.timestamp` 命令必须在 finalize 块中调用。
返回类型为 `i64`。


    assert.eq block.timestamp 1767976339i64;

#### network.id

`network.id` 命令返回程序执行所在网络的 ID。
这对于管理特定于网络的程序行为非常有用。
`network.id` 命令必须在 finalize 块中调用。

当前支持的网络 ID 为：

  * 0: 主网
  * 1: 测试网
  * 2: 金丝雀网


#### rand.chacha

`rand.chacha` 命令返回由 [ChaCha20 算法](<http://cr.yp.to/chacha.html>) 生成的随机数。
此命令支持采样随机的 `address`、`boolean`、`field`、`group`、`i8`、`i16`、`i32`、`i64`、`i128`、`u8`、`u16`、`u32`、`u64`、`u128` 和 `scalar` 类型。
最多可为 `rand.chacha` 命令提供两个额外的种子。
目前仅支持 ChaCha20，但将来可能支持其他随机数生成器。


    rand.chacha into r0 as field;
    rand.chacha r0 into r1 as field;
    rand.chacha r0 r1 into r2 as field;

#### 承诺

Aleo 指令支持以下将数据承诺为标准类型的语法。
请注意，`commit` 命令需要任意类型作为第一个参数，`scalar` 作为第二个参数。


    commit.bhp256 r0 r1 into r2 as address;
    commit.bhp256 r0 r1 into r2 as field;
    commit.bhp256 r0 r1 into r2 as group;
    commit.bhp512 ...;
    commit.bhp768 ...;

    commit.ped64 ...;
    commit.ped128 ...;

有关所支持承诺算法的完整列表，请查看 [Aleo 指令操作码](../opcodes/index.mdx)。

#### snark.verify、snark.verify.batch

`snark.verify` 命令在 finalize 作用域中验证 Varuna SNARK 证明，返回一个 `bool`。`snark.verify.batch` 命令使用单个批量证明批量验证多个证明。这两个命令都是**仅在 finalize 中使用** 。


    snark.verify r0 r1 r2 into r3;       // r0: vk ([u8]), r1: inputs ([field]), r2: proof ([u8]), r3: bool
    snark.verify.batch r0 r1 r2 into r3; // r0: vks, r1: nested inputs, r2: batch proof, r3: bool

`snark.verify.batch` 受编译时限制：最多 32 个电路，最多 128 个总实例。

查看 [Aleo 指令操作码](../opcodes/index.mdx#snarkverify) 获取完整规范。

#### position、branch.eq、branch.neq

`position` 命令（例如 `position exit`）指示跳转执行到的位置。
`branch.eq` 命令（例如 `branch.eq r0 r1 to exit`），如果 `r0` 和 `r1` 相等，则跳转执行到 `exit` 指示的位置。
`branch.neq` 命令（例如 `branch.neq r0 r1 to exit`），如果 `r0` 和 `r1` 不相等，则跳转执行到 `exit` 指示的位置。

**示例** 如果输入为 0u8，则 finalize 块成功退出；否则失败。


    program test_branch.aleo;

    function run_test:
        input r0 as u8.public;
        finalize r0;

    finalize run_test:
        input r0 as u8.public;
        branch.eq r0 0u8 to exit;
        assert.eq true false;
        position exit;

## 动态分派

动态分派使程序能够在运行时调用动态确定的函数。这解锁了接口模式、可插拔库和可组合协议，同时保留了 Aleo 的安全和隐私保障。有关包括转换电路和安全考虑在内的完整规范，请参阅 [ARC-0009](<https://vote.aleo.org/p/arc-0009>)。

### call.dynamic

`call.dynamic` 指令调用一个在运行时确定目标的函数。

**语法：**


    call.dynamic <PROG> <NET> <FUN> with <INPUTS> (as <INPUT_TYPES>) into <OUTPUTS> (as <OUTPUT_TYPES>);

标识符可以写作带引号的字面量（`'credits'`、`'aleo'`、`'transfer_public'`），它们被视为域元素。也接受类型为 `field` 的变量。


    program token_router.aleo;

    // 将转账路由到任何兼容的代币程序
    function route_transfer:
        input r0 as field.public;          // 代币程序名称
        input r1 as address.public;        // 接收者
        input r2 as u64.public;            // 金额
        call.dynamic r0 'aleo' 'transfer' with r1 r2 (as address.public u64.public) into r3 (as future.dynamic);
        async route_transfer r3 into r4;
        output r4 as token_router.aleo/route_transfer.future;

    finalize route_transfer:
        input r0 as future.dynamic;
        await r0;

**允许的类型：**

位置| 允许| 不允许
---|---|---
输入| 明文类型、`record.dynamic`| `record`、外部 record、`future`、`future.dynamic`
输出| 明文类型、`record.dynamic`、`future.dynamic`| `record`、外部 record、`future`

**限制：**

  * `call.dynamic` 只能出现在函数体中，不能出现在 finalize 块中
  * 目标程序、网络和函数是**公开** 的电路输入。不要使用私密数据来选择调用目标，因为目标是可观察的
  * 动态目标不能在运行时从链上映射中解析；调用方将目标作为输入提供（从链下查询），finalize 逻辑可以根据存储的值验证它们


**类型转换：** 当将 `record.dynamic` 传递给期望静态记录的调用方（反之亦然）时，AVM 会自动运行转换电路以证明两种表示之间的一致性。有关完整的电路细节，请参阅 [ARC-0009](<https://vote.aleo.org/p/arc-0009>)。

### get.record.dynamic

通过验证一个 Merkle 证明（证明条目以指定类型存在），从 `record.dynamic` 中检索命名字段。


    get.record.dynamic r0.microcredits into r1 as u64;
    get.record.dynamic r0.metadata into r2 as [u8; 32u32];

如果字段不存在或类型不兼容，Merkle 证明将失败且执行回滚。这强制执行隐式记录接口：任何包含所需字段的记录无论由哪个程序定义，都能通过检查。

`owner` 字段可以直接访问，无需 Merkle 验证：`r0.owner`。

### 动态映射操作

三个 finalize 作用域命令提供了对动态指定程序中映射的运行时访问：


    get.dynamic <PROG> <NET> <MAP>[<KEY>] into <DEST> as <TYPE>;
    get.or_use.dynamic <PROG> <NET> <MAP>[<KEY>] <DEFAULT> into <DEST> as <TYPE>;
    contains.dynamic <PROG> <NET> <MAP>[<KEY>] into <DEST>;

这些命令的行为与静态对应命令完全相同，区别在于程序和映射在运行时从域元素操作数中解析。
