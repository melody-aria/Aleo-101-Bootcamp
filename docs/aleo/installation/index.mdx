# Installation

## 概述

有两种常见的在本地使用 Aleo 进行构建的方法：

  * 如果您想从命令行**编写、构建和运行 Aleo 程序** （推荐大多数开发者使用），请安装 **Leo CLI** 。
  * 如果您想在 Rust 项目中**集成 Aleo 原语** （作为库或从源代码构建），请使用 **snarkVM** 。


## 安装 Leo CLI

按照官方的 Leo CLI 安装指南操作：

  * [安装 | 欢迎访问 Leo 文档](<https://docs.leo-lang.org/getting_started/installation>)


安装完成后，验证 Leo 是否可用：


    leo --version

## 安装并使用 snarkVM 作为 Rust 库

### 1\. 安装前置依赖

#### 1.1 安装 Git

**[bit.ly/start-git](<https://bit.ly/start-git>)**

#### 1.2 安装 Rust

我们建议使用以下方式安装 Rust： [rustup](<https://rustup.rs/>):

**macOS 或 Linux：**


    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

**Windows：**

下载 [Windows 64 位可执行文件](<https://win.rustup.rs/x86_64>) 或 [Windows 32 位可执行文件](<https://win.rustup.rs/i686>) 并按照屏幕上的说明操作。

#### 1.3 安装其他依赖项（macOS）

如果您在 macOS 上从源代码构建 Rust crate，可能需要以下包：


    brew install pkgconf
    brew install openssl

#### 1.4 检查前置依赖


    git --version
    cargo --version

### 2\. 将 snarkVM 添加到 Rust 项目

将 snarkVM 添加到您的 `Cargo.toml` 中：


    [dependencies]
    snarkvm = "<version>"

将 `<version>` 替换为您从以下地址选择的版本： [crates.io](<https://crates.io/crates/snarkvm>)。

### 3.（可选）从源代码构建 snarkVM

从源代码构建 snarkVM 是可选的，主要适用于您正在开发或审计 snarkVM 的情况。


    # 获取仓库的 staging 分支
    git clone --branch staging --single-branch https://github.com/ProvableHQ/snarkVM.git
    cd snarkVM

    # 以 release 模式构建
    cargo build --release
