# 欢迎

## 贡献指南

欢迎来到 Aleo 开发者文档。Aleo 秉承开源与去中心化原则，积极欢迎并珍视来自社区的各类贡献。我们鼓励所有形式的贡献，包括建议新主题和新内容，或提供示例以提高清晰度和实用性。

## 引言

Aleo 是一个完全开源、去中心化、可编程且保护隐私的 Layer 1 区块链，允许开发者创建内置隐私功能的应用。通过零知识证明，部署在 Aleo 上的程序能够在保持可验证性的同时使敏感数据保密。该平台提供高可扩展性，能够支持大规模应用，并提供不受限的运行环境，消除了传统区块链的种种限制。这种隐私、可扩展性和灵活性的独特结合，使 Aleo 成为构建下一代以隐私为核心的去中心化应用的理想平台。

更多详情，请访问[概念](../../../concepts/fundamentals/accounts/index.mdx)章节，探索 Aleo 的架构与特性。

## 帮助改进我们的生态系统

### 提交 ARC 提案

有关于开发标准或协议改进的想法？请按以下步骤提交 Aleo 征求意见稿（ARC）： 要创建新的 ARC 提案：

  1. 在 [GitHub 讨论区](<https://github.com/AleoHQ/ARCs/discussions/categories/arcs>) 提交您的提案，使用 [ARC-0000](<https://github.com/ProvableHQ/ARCs/tree/master/arc-0000>) 模板和一个可用的 ARC 编号。
  2. 在新子目录中提交包含您提案的 [拉取请求](<https://github.com/AleoHQ/ARCs/pulls>)。


要更新现有 ARC：

  1. 提交一个包含您修改的 拉取请求。
  2. 如果修改较为重大，您可能需要重新提交一个全新的 ARC。


#### ARC 推进流程

ARC 将以"草案"状态启动，并按以下阶段推进：

提案提交后：

  1. 社区将讨论和评审该提案。维护者将关注此 ARC，待其准备就绪后将其状态更改为"活跃"。
     * ARC 将根据投票数量和是否存在原型来确定优先级。
     * ARC 将在特定的社区会议中进行讨论。提案者将有机会参与并加入讨论。
     * 在此阶段之前，提案者可以撤回 ARC，或者若长时间无活动，维护者也可将其撤回。
  2. Aleo 网络基金会（ANF）的治理者或团队成员将在 Aleo 治理平台（<https://vote.aleo.org/>）上创建正式提案并启动投票流程。
  3. 社区将对提案进行投票表决。
  4. 如果提案被接受，其状态将更新为"已接受"，相关的 拉取请求 将被合并到 ARCs 仓库中。若提案被拒绝，状态将恢复为"草案"。
  5. 相关方应完成实施工作。可根据需要通过新的 PR 对 ARC 进行更新，无需再次投票。
  6. 实施完成后，根据提案的性质，状态将从"已接受"更改为"最终"或"活跃"。相关的讨论将被关闭。


如果提案被新提案替代，则其状态可能为"已弃用"。

#### 状态说明

有关状态的详细说明，请参阅 [ARC-0001](<https://github.com/ProvableHQ/ARCs/tree/master/arc-0001>)。

### 为 Aleo 做贡献

构成 Aleo 的每一个组件都是开源的，欢迎各种形式的贡献，包括[本文档](../documentation_contribute/index.mdx)。每个组件都有其专属的贡献指南，为方便参考，汇总如下：

  * [DevDocs](../documentation_contribute/index.mdx) \- 本开发者文档
  * [Leo](<https://github.com/ProvableHQ/leo-docs-source>) \- 基于 Rust 的 DSL，用于在 Aleo 上编写程序
  * [Provable SDK](<https://github.com/ProvableHQ/sdk/tree/mainnet/sdk>) \- 用于创建零知识应用的 Javascript/Typescript 工具
  * [Create-leo-app](<https://github.com/ProvableHQ/sdk/tree/mainnet/create-leo-app>) \- 搭建您的第一个 Aleo 项目
  * [Aleo 指令](../../aleo/aleo/index.mdx) \- Leo 语言编译成的中间表示
  * [SnarkOS](../snarkos_contribute/index.mdx) \- Aleo 的操作系统或区块链节点客户端
  * [SnarkVM](../snarkvm_contribute/index.mdx) \- 执行可验证计算的虚拟机
