# DevDocs Contribution

本指南将引导您如何有效地为我们的文档做出贡献。

## 提供反馈

如果你发现了问题或有改进建议但没有时间亲自修改，您仍然可以通过 GitHub Issues 提供反馈：

  1. 前往 [Issues](<https://github.com/AleoNet/welcome/Issues>) welcome 仓库的页面。
  2. 点击 "新建 Issue".
  3. 如果有合适的 Issue 模板请选择，否则从空白 Issue 开始。
  4. 提供清晰简洁的标题，概括该问题或建议.
  5. 在描述中:
     * 对于错误或缺陷：描述你期望看到的内容和你实际看到的内容。 如适用，请包含复现步骤。
     * 对于改进建议：解释你的想法以及它将如何改进文档。
     * 如相关，请包含截图或代码片段来说明你的观点。
  6. 为你的 Issue 添加适当的标签（如 "文档"、"增强"、"缺陷"）.
  7. 提交 Issue.


你的反馈帮助我们识别需要改进的领域，确保文档对所有人都是准确且有帮助的。

## 快速入门

  1. 复刻 [welcome](<https://github.com/AleoNet/welcome>) 仓库在 GitHub 上.

  2. 在本地克隆你复刻的仓库:

         git clone https://github.com/YOUR-USERNAME/welcome.git

  3. 进入项目目录:

         cd welcome

  4. 在文档目录中进行修改:

         cd documentation

这里你可以找到要编辑的内容文件。


## 进行更改

  1. 为你的更改创建一个新分支:

         git checkout -b your-feature-branch

  2. 对文档文件进行修改. 大多数内容是 Markdown 格式的.
  3. 使用描述性的提交信息提交你的更改:

         git commit -am "Add description of your changes"


## 提交你的贡献

  1. 将你的更改推送到你复刻的仓库:

         git push origin your-feature-branch

  2. 前往 [welcome](<https://github.com/AleoNet/welcome>) 仓库在 GitHub 上，并从 `your-feature-branch` 创建一个新的 pull request。
  3. 为你的 pull request 提供清晰的标题和描述，解释你所做的更改。
  4. 提交 pull request 供审查。


## 指南

  * 确保你的写作清晰、简洁，并遵循文档现有的风格.
  * 对标题、代码块和其他元素使用正确的 Markdown 格式.
  * 如果要添加新页面，适当更新侧边栏配置.
  * 包含任何必要的图片或图表来说明概念.
  * 验证所有链接都是正确且有效的.


## 需要帮助?

如果你有任何问题或需要帮助，请随时在 [welcome](<https://github.com/AleoNet/welcome>) 仓库中发起 Issue 或联系维护者.

感谢你为 Aleo 文档做出贡献!
