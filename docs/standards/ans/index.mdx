# Aleo 名称服务

## 概述

Aleo 名称服务（ANS）是一个标准程序，用于在 Aleo 区块链上管理人类可读的域名。该标准源自 [ARC-137 提案](<https://github.com/ProvableHQ/ARCs/discussions/45>)，并通过社区投票在 [Aleo 治理](<https://vote.aleo.org/p/137>)上正式获批。

ANS 旨在通过允许可记忆和可更新的、人类可读的标识符来简化与 Aleo 资源的交互。它同时支持公开和私有域名，各自服务于不同的用例和隐私需求。公开域名提供稳定的人类可读标识符，可用于指定网络资源，而私有域名则可以在不暴露接收者真实 Aleo 地址的情况下进行 Aleo Credits（ACs）的私人转账。

### 与 ARC-0721 的兼容性

虽然 ANS 旨在兼容 [ARC-0721 标准](../nft_standards/index.mdx)，但它需要对 NFT 结构进行一些修改以支持其独特的域名注册功能。这些差异对于在保持与更广泛 Aleo 生态系统的兼容性的同时，实现域名标识符的动态创建是必需的。

#### NFT 结构的差异

其中一个关键区别在于 ANS 中使用的 NFT 结构。虽然 ARC-0721 为 NFT 定义了标准结构，但 ANS 需要对 NFT 记录中的 data 字段采用动态方式。这是由于域名注册的性质所致，每个 NFT 必须反映一个唯一的标识符（name_hash），该标识符仅在用户注册域名时才确定。以下是 ANS 特定的 NFT 结构：


    // The ANS NFT structure diverges from ARC-0721 in the `data` field.
    // Here, `data` is not predefined but is dynamically created based on the domain name registered by the user.
    // This `data` serves as the name_hash of the name, uniquely identifying the domain within ANS.
    record NFT {
        owner: address,
        data: data,
        edition: scalar // The edition number, similar to ARC-0721's structure.
    }

#### 拥抱差异以实现增强功能

对 ANS 中 NFT 结构的修改对于支持协议的功能和目标是必要的。虽然 ANS 努力与现有的 ARC-0721 等标准保持一致，但它也认识到需要创新并调整其 NFT 表示方式，以有效地实现其独特目标。这种方法确保 ANS 能够提供一个强大且以隐私为中心的命名服务，为更广泛的 Aleo 生态系统提供补充。

社区和开发者了解这些差异非常重要，以实现无缝集成并在适用的情况下充分利用两个标准的优势。

本文档概述了 Aleo 名称服务程序的功能并提供了使用指南。原始源代码可在 [GitHub 仓库](<https://github.com/S-T-Soft/ans-programs>)中找到。

## ANS 组件

Aleo 名称服务由三个主要组件组成：

  1. 注册表程序：管理域名系统，将名称映射到解析器并允许更新这些映射
  2. 注册商：处理域名分配和顶级域管理
  3. 解析器：处理资源查询并返回请求的数据


## 注册表程序

注册表程序是 ANS 的核心组件，负责管理域名系统并维护名称与其关联数据之间的映射。

### 数据结构

#### Name 结构体


    struct Name {
        name: [u128; 4],
        parent: field // The name_hash of the parent name, for top level domain(tld), this attribute is 0field.
    }

#### NameStruct


    struct NameStruct {
        name: [u128; 4],
        parent: field,
        resolver: field // The resolver program address
    }

#### Data 结构体


    struct data {
        metadata: [field; 4], // the first element is the name_hash of the name
    }

#### NFT 记录


    record NFT {
        owner: address,
        data: data,
        edition: scalar
    }

#### NFTView 记录


    record NFTView {
        owner: address,
        data: data,
        edition: scalar,
        is_view: bool
    }

### 映射


    mapping nft_owners: field => address;
    mapping names: field => NameStruct;
    mapping tlds: field => [u128; 4];
    mapping primary_names: address => field;
    mapping name_versions: field => u64;
    mapping resolvers: ResolverIndex => [u128; 4];
    mapping domain_credits: field => u64;

### 核心函数

注册表程序提供以下核心功能：

  * `register_tld()`: 注册一个新的顶级域
  * `register()`: 注册一个新的域名
  * `register_private()`: 注册一个私有子域
  * `register_public()`: 注册一个公开域
  * `transfer_private()`: 转移私有域的所有权
  * `transfer_public()`: 转移公开域的所有权
  * `set_primary_name()`: 将域设为地址的主要名称
  * `unset_primary_name()`: 移除主要名称设置
  * `set_resolver()`: 为域设置解析器地址


## 注册商

注册商负责管理域名分配和顶级域（TLD）操作。它们与注册表程序协同工作，处理域名注册和管理。

### 注册商函数

  * 域名验证
  * TLD 管理
  * 注册费用处理
  * 域名续期处理


## 解析器

解析器是专门处理资源查询并返回域请求数据的程序。它们处理域名到其关联资源的实际解析。

### 解析器函数

  * 资源查询处理
  * 数据解析
  * 记录管理
  * 版本控制


### ResolverIndex 结构体


    struct ResolverIndex {
        name: field, // The name_hash of the domain
        category: u128, // The type of the resolver
        version: u64 // The version of the resolver
    }

### 解析器操作

  * `set_resolver_record()`: 为私有域设置解析器记录
  * `unset_resolver_record()`: 移除私有域的解析器记录
  * `set_resolver_record_public()`: 为公开域设置解析器记录
  * `unset_resolver_record_public()`: 移除公开域的解析器记录


## 隐私信用转账方案

隐私信用转账方案是建立在 Aleo 名称服务（ANS）之上的一项创新功能，支持信用的私人转账。该方案确保交易双方都不需要披露其 Aleo 地址，从而在保持易用性的同时增强了隐私性。

### 转账信用

此功能支持向 ANS 域转账信用而无需暴露接收者的真实 Aleo 地址。它涉及一个秘密，允许域名持有者私下领取被转账的信用。

#### 向 ANS 名称转账私有信用


    // Function for transferring credits to an ANS domain without revealing the domain holder's real address.
    // @param receiver: The name_hash of the recipient ANS domain.
    // @param secret: The secret associated with the transaction, used for claim verification.
    // @param amount: The amount of credits to be transferred.
    // @param pay_record: The record of the payment being made.
    fn transfer_credits(receiver: field, secret: [u128; 2], amount: u64, pay_record: credits.aleo::credits)

#### 向 ANS 名称转账公开信用


    // Function for transferring credits to an ANS domain without revealing the domain holder's real address.
    // @param receiver: The name_hash of the recipient ANS domain.
    // @param secret: The secret associated with the transaction, used for claim verification.
    // @param amount: The amount of credits to be transferred.
    fn transfer_credits_public(receiver: field, secret: [u128; 2], amount: u64)

### 领取信用

这些函数允许域名持有者领取被转账的信用。根据域是公开还是私有，将使用相应的领取函数。

#### 领取信用（私有）


    // Function for a domain holder to claim credits using a private ANS domain represented by an NFT and a secret.
    // This ensures that the claim process remains private and the domain holder's real address is not exposed.
    // @param receiver: The receiver address
    // @param nft: The NFT record representing the private ANS domain.
    // @param secret: The secret used to verify the claim.
    // @param amount: The amount of credits to be claimed.
    fn claim_credits_private(receiver: address, nft: NFT, secret: [u128; 2], amount: u64)

#### 领取信用（公开）——调用者是 ANS 名称的所有者


    // Similar to the private claim function, this enables the claiming of credits for a public ANS domain.
    // The domain holder uses a secret to claim the credits, maintaining privacy.
    // @param receiver: The receiver address
    // @param name_hash: The name_hash of the public ANS domain.
    // @param secret: The secret used to verify the claim.
    // @param amount: The amount of credits to be claimed.
    fn claim_credits_public(receiver: address, name_hash: field, secret: [u128; 2], amount: u64)

#### 以签名者身份领取信用——签名者是 ANS 名称的所有者


    // Similar to the private claim function, this enables the claiming of credits for a public ANS domain.
    // The domain holder uses a secret to claim the credits, maintaining privacy.
    // @param receiver: The receiver address
    // @param name_hash: The name_hash of the public ANS domain.
    // @param secret: The secret used to verify the claim.
    // @param amount: The amount of credits to be claimed.
    fn claim_credits_as_signer(receiver: address, name_hash: field, secret: [u128; 2], amount: u64)
