# Token Registry 程序


::: warning[Not Recommended for New Projects]
The Token Registry Program remains live on mainnet and continues to serve existing integrations. However, it is **not recommended for new projects**. Newer token standards are currently under active community discussion and are expected to supersede the Token Registry:

  * [ARC-20](<https://github.com/ProvableHQ/ARCs/discussions/124>) — 提议的 Aleo 同质化代币标准
  * [ARC-22](<https://github.com/ProvableHQ/ARCs/discussions/125>) — 提议的符合监管/受监管的 Aleo 同质化代币标准，扩展 ARC-20


新项目应该关注这些讨论，并在标准最终确定后考虑基于新标准进行构建。
:::

## 概述

Token Registry 程序是一个用于在 Aleo 上发行和管理代币的单例程序。与为每个代币部署单独的程序不同，所有代币都在这个中央注册表中注册，由注册表管理其余额。DeFi 程序仅依赖注册表，因此可以添加新代币而无需重新部署现有的 DeFi 程序。作为次要优势，注册表内的私有转账隐藏了正在转账的具体代币，从而改善了匿名集。

此设计早于 [动态分发（ARC-0009）](<https://github.com/ProvableHQ/ARCs/tree/master/arc-0009>)。随着动态分发的最终确定，程序可以在运行时调用其他程序而无需编译时导入，从而实现了 [ARC-20](<https://github.com/ProvableHQ/ARCs/discussions/124>) 中描述的按程序构建的代币模型。Token Registry 仍在主网上运行并继续为现有集成服务——有关新项目的指导，请参阅上方的警告。

该标准源自社区讨论和 [ARC-21 提案](<https://vote.aleo.org/p/21>)的批准。

原始源代码可在此处[查看](<https://github.com/demox-labs/aleo-standard-programs/blob/main/token_registry/src/main.leo>)。

## 如何使用 Token Registry 程序

使用唯一的代币 ID、名称、符号、小数位数和最大供应量调用 `register_token`。将 `external_authorization_required` 设置为 `true` 要求在代币可被使用之前获得 `external_authorization_party` 的批准，该方使用 `prehook_public` 或 `prehook_private` 为特定所有者解锁余额，并指定到期区块高度。管理员稍后可以通过 `update_token_management` 更新 `external_authorization_party`。

一旦注册，代币可以公开铸造（`mint_public`）或私有铸造（`mint_private`），以及公开销毁（`burn_public`）或私有销毁（`burn_private`）。如果调用者不是管理员，铸造和销毁需要 `MINTER_ROLE`/`BURNER_ROLE` 或 `SUPPLY_MANAGER_ROLE`。

所有者可以公开转账（`transfer_public`）或私下转账（`transfer_private`），并可以使用 `transfer_public_to_private` 和 `transfer_private_to_public` 在公开和私有余额之间进行转换。

## 数据结构

### Token 记录


    record Token {
      owner: address,
      amount: u128,
      token_id: field,
      external_authorization_required: bool,
      authorized_until: u32           // block height until authorization expires
    }

### TokenMetadata 结构体


    struct TokenMetadata {
      token_id: field,
      name: u128,                          // ASCII text as u128 bitstring
      symbol: u128,                        // ASCII text as u128 bitstring
      decimals: u8,
      supply: u128,
      max_supply: u128,
      admin: address,
      external_authorization_required: bool,
      external_authorization_party: address
    }

### TokenOwner 结构体


    struct TokenOwner {
      account: address,
      token_id: field
    }

### Balance 结构体


    struct Balance {
      token_id: field,
      account: address,
      balance: u128,
      authorized_until: u32
    }

### Allowance 结构体


    struct Allowance {
      account: address,
      spender: address,
      token_id: field
    }

## 映射

映射| 键| 值
---|---|---
`registered_tokens`| `field` (token ID)| `TokenMetadata`
`balances`| `field` (hash of token ID + account)| `Balance`
`allowances`| `field` (hash of token ID + account + spender)| `Allowance`
`roles`| `field` (hash of token ID + account)| `u8`

## 常量

常量| 值| 描述
---|---|---
`CREDITS_RESERVED_TOKEN_ID`| `3443843282313283355522573239085696902919850365217539366784739393210722344986field`| ALEO credits 的保留代币 ID
`MINTER_ROLE`| `1u8`| 铸造角色
`BURNER_ROLE`| `2u8`| 销毁角色
`SUPPLY_MANAGER_ROLE`| `3u8`| 铸造和销毁角色

## 函数

### `initialize()`

使用硬编码的元数据注册 ALEO credits 代币（名称/符号 "credits"，6 位小数，最大供应量 10 quadrillion）。参数已硬编码以防止抢先交易。将 `wrapped_credits.aleo` 设置为管理员并禁用外部授权，因此之后无法修改元数据。

* * *

### `register_token()`

注册新代币。调用者成为管理员。

参数| 类型| 描述
---|---|---
`token_id`| `public field`| 唯一代币标识符
`name`| `public u128`| 代币名称
`symbol`| `public u128`| 代币符号
`decimals`| `public u8`| 小数位数
`max_supply`| `public u128`| 最大供应量
`external_authorization_required`| `public bool`| 是否需要外部授权
`external_authorization_party`| `public address`| 授权方的地址

返回值：`Future`

* * *

### `update_token_management()`

更新代币的管理员和外部授权方。只能由当前管理员调用。

参数| 类型| 描述
---|---|---
`token_id`| `public field`| 代币标识符
`admin`| `public address`| 新管理员地址
`external_authorization_party`| `public address`| 新授权方地址

返回值：`Future`

* * *

### `set_role()` / `remove_role()`

`set_role` 为给定代币的账户分配角色（`MINTER_ROLE`、`BURNER_ROLE` 或 `SUPPLY_MANAGER_ROLE`）。`remove_role` 清除角色。两者都要求调用者是代币管理员。

参数| 类型| 描述
---|---|---
`token_id`| `public field`| 代币标识符
`account`| `public address`| 目标账户
`role`| `public u8`| 角色值（仅 `set_role`）

返回值：`Future`

* * *

### `mint_public()` / `mint_private()`

向接收者铸造代币。需要管理员、`MINTER_ROLE` 或 `SUPPLY_MANAGER_ROLE`。

参数| 类型| 描述
---|---|---
`token_id`| `public field`| 代币标识符
`recipient`| `address`（`mint_private` 为私有）| 接收者地址
`amount`| `public u128`| 铸造数量
`external_authorization_required`| `public bool`| （仅 `mint_private`）
`authorized_until`| `public u32`| 授权到期区块高度

返回值：`mint_public` → `Future`；`mint_private` → `Token`、`Future`

* * *

### `burn_public()` / `burn_private()`

销毁代币。需要管理员、`BURNER_ROLE` 或 `SUPPLY_MANAGER_ROLE`。

参数| 类型| 描述
---|---|---
`token_id`| `public field`| 代币标识符（仅 `burn_public`）
`owner`| `public address`| 代币所有者（仅 `burn_public`）
`input_record`| `Token`| Token 记录（仅 `burn_private`）
`amount`| `public u128`| 销毁数量

返回值：`burn_public` → `Future`；`burn_private` → 剩余的 `Token`、`Future`

* * *

### `prehook_public()` / `prehook_private()`

由 `external_authorization_party` 调用，为所有者解锁指定数量的代币，有效期至给定区块高度。

参数| 类型| 描述
---|---|---
`owner`| `public address`| 代币所有者（仅 `prehook_public`）
`input_record`| `Token`| Token 记录（仅 `prehook_private`）
`amount`| `u128`| 授权金额
`authorized_until`| `u32`| 授权到期区块高度

返回值：`prehook_public` → `Future`；`prehook_private` → 未授权的 `Token`、已授权的 `Token`、`Future`

* * *

### `transfer_public()`

在公开余额之间转账代币。调用者是发送者（`self.caller`）。

参数| 类型| 描述
---|---|---
`token_id`| `public field`| 代币标识符
`recipient`| `public address`| 接收者地址
`amount`| `public u128`| 转账金额

返回值：`Future`

* * *

### `transfer_public_as_signer()`

与 `transfer_public` 相同，但使用 `self.signer` 作为发送者，从而可以在任意程序调用链中使用。

参数和返回值与 `transfer_public` 相同。

* * *

### `approve_public()` / `unapprove_public()`

`approve_public` 授予支出方代表所有者转账代币的授权额度。`unapprove_public` 撤销或减少该授权额度。

参数| 类型| 描述
---|---|---
`token_id`| `public field`| 代币标识符
`spender`| `public address`| 支出方地址
`amount`| `public u128`| 要授权或取消授权的金额

返回值：`Future`

* * *

### `transfer_from_public()`

使用预先批准的授权额度从所有者向接收者转账代币。

参数| 类型| 描述
---|---|---
`token_id`| `public field`| 代币标识符
`owner`| `public address`| 代币所有者
`recipient`| `public address`| 接收者地址
`amount`| `public u128`| 转账金额

返回值：`Future`

* * *

### `transfer_public_to_private()`

将公开余额转换为私有代币记录。

参数| 类型| 描述
---|---|---
`token_id`| `public field`| 代币标识符
`recipient`| `address`（私有）| 接收者地址
`amount`| `public u128`| 要转换的金额
`external_authorization_required`| `public bool`| 是否需要授权

返回值：`Token`、`Future`

* * *

### `transfer_from_public_to_private()`

使用预先批准的授权额度代表所有者将公开余额转换为私有代币记录。

参数| 类型| 描述
---|---|---
`token_id`| `public field`| 代币标识符
`owner`| `public address`| 代币所有者
`recipient`| `address`（私有）| 接收者地址
`amount`| `public u128`| 要转换的金额
`external_authorization_required`| `public bool`| 是否需要授权

返回值：`Token`、`Future`

* * *

### `transfer_private()`

在私有记录之间转账代币。

参数| 类型| 描述
---|---|---
`recipient`| `address`（私有）| 接收者地址
`amount`| `u128`| 转账金额
`input_record`| `Token`| 发送者的 Token 记录

返回值：剩余的 `Token`（发送者）、接收的 `Token`（接收者）、`Future`

* * *

### `transfer_private_to_public()`

将私有代币记录转换为公开余额。

参数| 类型| 描述
---|---|---
`recipient`| `public address`| 接收者地址
`amount`| `public u128`| 要转换的金额
`input_record`| `Token`| 要消费的 Token 记录

返回值：剩余的 `Token`、`Future`

* * *

### `join()`

将同一代币的两个私有代币记录合并为一个。总金额不变。

参数| 类型| 描述
---|---|---
`token_1`| `private Token`| 第一个 Token 记录
`token_2`| `private Token`| 第二个 Token 记录

返回值：合并后的 `Token`

* * *

### `split()`

将一个私有代币记录拆分为两个。总金额不变。

参数| 类型| 描述
---|---|---
`token`| `private Token`| 要拆分的 Token 记录
`amount`| `private u128`| 拆分到第一个输出中的金额

返回值：拆分后的 `Token`、剩余的 `Token`
