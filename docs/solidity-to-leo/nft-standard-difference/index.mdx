# NFT 标准差异

## 介绍

ERC-721 是以太坊上非同质化代币的标准，但每一份状态（所有者、元数据、价格）都是公开的。

ARC-721 是 Aleo 上以 Leo 实现的变体，保持相同的易用性，同时允许您选择哪些部分保持私有，哪些存储在链上。该标准源自 [ARC-721 proposal](<https://github.com/ProvableHQ/ARCs/discussions/79>) 并经 [Aleo Governance](<https://vote.aleo.org/p/721>) 社区投票正式批准。该标准利用 Aleo 独特的隐私特性，提供了比 ERC-721 更强大的功能。例如，ARC-721 中的每个 NFT 都有两个独立的隐私控制：一个用于所有权，一个用于 NFT 数据本身，两者都可配置隐私设置。有关 NFT 标准和实现细节的更多信息，请参阅 [NFT 标准文档](../../standards/nft_standards/index.mdx)。

Aleo 上的程序可组合性历来需要中心注册表方式（类似于 [Token Registry Program](../../standards/token_registry/index.mdx)（类似于同质化代币的 [代币注册表程序](../../standards/token_registry/index.mdx)）。随着 [动态调度（ARC-0009）](<https://github.com/ProvableHQ/ARCs/tree/master/arc-0009>) 现已最终确定，程序可以在运行时调用其他程序而无需编译时导入，为更灵活的 NFT 可组合性模式打开了大门。拟议的 [NFT 注册表程序（ARC-722）](<https://github.com/ProvableHQ/ARCs/discussions/80>) 将作为由 (registry_program_id, collection_id) 对标识的 NFT 集合的中心枢纽，但该提案仍处于提议阶段，尚未经过投票或批准。

## 快速对比

函数| **ERC-721 (Ethereum)**| **ARC-721 (Aleo)**
---|---|---
**所有者可见性**|  始终公开| 可选 — 私有 record _或_ 公开 mapping
**元数据可见性**|  您发布的任何内容都可见| 私有、公开或混合（混合链上/链下）
**唯一标识符**|  递增 `tokenId uint256`| 承诺 `field = hash(data) ⊕ edition`（不透明）
**转账函数**| `transferFrom`, `safeTransferFrom`| `transfer_private`, `transfer_private_to_public`, `transfer_public_as_signer`, `transfer_public_to_private`
**授权**| `approve`, `setApprovalForAll`| `approve_public`, `set_for_all_approval`
**私有销毁与重新铸造**|  不可能不留痕迹| `update_edition_private` 重新混淆承诺

## 架构差异

### 状态管理

**ERC-721**


    contract ERC721 {
        // 代币名称
        string private _name;

        // 代币符号
        string private _symbol;

        // 从代币 ID 到所有者地址的 mapping
        mapping(uint256 => address) private _owners;

        // 所有者地址到代币数量的 mapping
        mapping(address => uint256) private _balances;

        // Mapping from token ID to approved address
        mapping(uint256 => address) private _tokenApprovals;
    }

**ARC-721**

以下是 ARC-721 NFT 程序的数据结构示例，使用 record 作为私有存储，mapping 作为公开存储。struct 的名称不必与 `data` 和 `attribute` 匹配，允许导入多个 NFT 集合程序而不产生名称冲突：


    record NFT {
        private owner: address,   // 除非公开否则隐藏
        private data: data,      // 链上 struct（可映射链下 JSON）
        private edition: scalar,  // 作为混淆器在链上存储 NFT 承诺
    }

    // NFT 数据始终私有并存储在 NFTView 中，即使所有权已公开
    // is_view 始终为 true，以在 record 的明文表示中区分 NFTView 和 NFT
    record NFTView {
        private owner: address,
        private data: data,
        private edition: scalar,
        public is_view: bool
    }

    // Example attribute, optional
    struct attribute {
        trait_type: [field; 4],
        _value: [field; 4],
    }

    struct data {
        metadata: [field; 4], // URI of offchain metadata JSON
        // (optional) name: [field; 4],
        // (optional) image: [field; 16],
        // (optional) attributes: [attribute; 4],
        // (optional) ...
    }

    // On-chain NFT data storage if made public
    struct nft_content {
        data: data,
        edition: scalar
    }
    mapping nft_contents: field => nft_content; // commit(data, edition) => (data, edition)

    // Approval data structure
    struct Approval {
        collection_id: field,
        approver: address,
        spender: address,
    }
    mapping for_all_approvals: field => bool     // approval hash → bool
    mapping nft_approvals:     field => field    // commit → approval hash

### 字符串管理

由于 Leo 没有原生字符串类型，字符串使用 `field` 元素数组管理（如上面 [ARC-721 数据结构](#state-management) 中的 `attribute` 和 `data` struct 所示）。

Leo 中字符串管理的关键点：

  * 数组长度可根据所需的最大字符数调整
  * 使用 field 而非 u128，因为它们在相同约束下提供大约两倍的数据量
  * 这种方法对于在 NFT 中存储元数据 URI 和其他字符串数据特别有用
  * 对于 JavaScript/TypeScript 应用，示例 [utility](https://github.com/zsociety-io/aleo-standard-programs/blob/main/arc721/utils/strings.mdx) 可在 ARC-721 实现中使用，用于在 JavaScript 字符串和 Aleo 明文之间进行转换


### NFT 标识符

**ERC-721**

在 ERC-721 中，NFT 由简单的递增 `uint256 tokenId` 标识。

**ARC-721**

ARC-721 中使用 NFT commit 来标识每个唯一的 NFT：


    mapping nft_commits: field => bool; // NFT commit => NFT 存在或曾经存在

    fn commit_nft(
        nft_data: data,
        nft_edition: scalar
    ) -> field {
        let data_hash: field = BHP256::hash_to_field(nft_data);
        let commitment: field = BHP256::commit_to_field(data_hash, nft_edition);
        return commitment;
    }

### NFT 创建与结构

**ERC-721**

ERC-721 提供了一个标准的铸造函数，可选的扩展用于自定义功能。


    contract MyNFT is ERC721 {
        constructor(string memory name, string memory symbol) ERC721(name, symbol) {}

        function mint(address to, uint256 tokenId) public {
            _mint(to, tokenId);
        }
    }

**ARC-721**

ARC-721 提供了定义自定义 NFT 创建逻辑的灵活性，因为 NFT 程序由开发者部署。标准实现包含用于以不同隐私设置铸造 NFT 的函数：


    // 铸造私有 NFT
    // 返回表示所有权并包含 NFT 数据的 NFT record。
    fn mint_private(
        admin_nft: nft_records.aleo::NFT,
        private recipient: address,
        private nft_data: Data,
        private nft_edition: scalar,
    ) -> (nft_records.aleo::NFT, Final)

    // 铸造私有 NFT，在 finalization 时公开验证管理员权限。
    // 返回表示所有权并包含 NFT 数据的 NFT record。
    fn mint_private_as_public(
        private recipient: address,
        private collection_id: field,
        private nft_data: Data,
        private nft_edition: scalar,
    ) -> (nft_records.aleo::NFT, Final)

    // Mints a public NFT, with private admin as authorization.
    // Returns NFTView record that contains private NFT data, ownership is stored publicly on-chain.
    fn mint_public_as_private(
        admin_nft: nft_records.aleo::NFT,
        public recipient: address,
        private nft_data: Data,
        private nft_edition: scalar,
    ) -> (nft_records.aleo::NFTView, Final)

    // Mints a public NFT
    // Returns NFTView record that contains private NFT data, ownership is stored publicly on-chain.
    fn mint_public(
        public recipient: address,
        public collection_id: field,
        private nft_data: Data,
        private nft_edition: scalar,
    ) -> (nft_records.aleo::NFTView, Final)

    // Make a NFT data public
    fn publish_nft_content(
        public nft_data: Data,
        public nft_edition: scalar,
    ) -> Final

### 转账机制

**ERC-721**


    function transferFrom(address from, address to, uint256 tokenId) public
    function safeTransferFrom(address from, address to, uint256 tokenId) public

**ARC-721**


    // 私有转账
    fn transfer_private(
        private nft: NFT,
        private to: address,
    ) -> NFT

    // 从函数调用者（msg.sender）的公开转账
    fn transfer_public(
        private nft_data: data,
        private nft_edition: scalar,
        public to: address,
    ) -> (NFTView, Final)

    // 从交易签名者（tx.origin）的公开转账
    fn transfer_public_as_signer(
        private collection_id: field,
        private nft_data: Data,
        private nft_edition: scalar,
        public recipient: address,
    ) -> (NFTView, Final)

    // Public transfer by an approved spender
    fn transfer_from_public(
        public from: address,
        public to: address,
        private nft_data: data,
        private nft_edition: scalar,
    ) -> (NFTView, Final)

    // Convert private NFT ownership to public NFT ownership
    fn transfer_private_to_public(
        nft: nft_records.aleo::NFT,
        public recipient: address,
    ) -> (nft_records.aleo::NFTView, Final)

    // Convert public NFT ownership to private NFT ownership
    fn transfer_public_to_private(
        private nft_data: data,
        private nft_edition: scalar,
        private to: address,
    ) -> (NFT, Final)

    // Convert public NFT ownership to private NFT ownership by an approved sender
    fn transfer_from_public_to_private(
        private collection_id: field,
        public from: address,
        public recipient: address,
        private nft_data: Data,
        private nft_edition: scalar,
    ) -> (NFT, Final)

#### 转账流程并排对比

**转账流程**| **ERC-721 (Solidity)**| **ARC-721 (Leo)**| **隐私级别**
---|---|---|---
**私有 → 私有**|  ❌ 不支持| `transfer_private`| 完全私有 — 无公开痕迹
**私有 → 公开**|  ❌ 不支持| `transfer_private_to_public`| 所有者变为可见；内容在 `NFTView` 中保持隐藏
**公开 → 公开**| `transferFrom` / `safeTransferFrom`| `transfer_public`（通过 `self.caller`）`transfer_public_as_signer`（通过 `self.signer`）| 匹配 ERC-721 行为并具有额外的灵活性
**公开 → 私有**|  ❌ 不支持| `transfer_public_to_private` `transfer_from_public_to_private`| 所有权变为私有；资产从公开注册表中消失
**公开（由支出者）**| `transferFrom` 配合 `approve()`| `transfer_from_public` `transfer_from_public_to_private`| 已授权的代理转账，公开或私有

## 设置

ARC-721 还推荐了集合的标准设置：


    mapping general_settings: u8 => field;  // Setting index => Setting value

  * `0u8` \- 可铸造的 NFT 总数（所有版本）
  * `1u8` \- 可铸造的 NFT 总数（首发版）
  * `2u8` \- NFT 符号
  * `3u8` \- NFT 基础 URI，第 1 部分
  * `4u8` \- NFT 基础 URI，第 2 部分
  * `5u8` \- NFT 基础 URI，第 3 部分
  * `6u8` \- NFT 基础 URI，第 4 部分
  * `7u8` \- 管理员地址哈希


## 版本

ARC-721 引入了 edition 字段作为每个 NFT record 中的强制性 scalar。使用 edition 的原因有三个：

**隐私盐值** — edition 与 NFT 数据的 BHP256 哈希混合，生成 `nft_commit = BHP256::commit_to_field(BHP256::hash_to_field(data), edition)`。 此盲化因子可防止尝试判断两个承诺是否包含相同底层数据的暴力攻击。

**唯一性锚点** — 因为承诺依赖 edition，任何更改（即使数据相同）都会生成一个全新的 nft_commit，确保每个代币都是不可替代的。

**重新混淆** — 所有者可以通过选择一个新的随机 scalar 并调用 `update_edition_private` 来"轮换"隐私，断开与先前转账的链上关联。

## 发布（和重新隐藏）内容

当 NFT 的数据应变为公开时，所有者调用：


    fn publish_nft_content(nft_data, nft_edition)

它将明文 struct 复制到：


    mapping nft_contents: field => nft_content;

如果以后需要再次隐藏它：

  1. 将 NFT 通过 `transfer_public_to_private` 转为 **私有** 。
  2. 调用 `update_edition_private` 滚动承诺。


## 隐私特性

ARC-721 标准提供了几个增强隐私的特性：

  1. **私有所有权**

     * NFT 可以使用 Aleo record 私有持有
     * 可以在不透露所有者身份的情况下验证所有权
     * 支持私有和公开所有权状态之间的转换
  2. **私有数据**

     * NFT 数据默认保持私有
     * 通过 `publish_nft_content` 可选的数据公开
  3. **基于版本的隐私**

     * 每个 NFT 有唯一的 edition 号码（scalar）
     * Editions 在保持唯一性的同时实现隐私
     * 支持 edition 更新以进行重新混淆
  4. **灵活的可见性**

     * `NFTView` record 类型用于公开所有权和私有数据
     * `NFT` record 类型用于完全私有的所有权
     * 状态之间内置的转换函数
