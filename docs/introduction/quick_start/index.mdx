# Quick Start

Welcome to the Aleo Quickstart. This quickstart will guide you through the core concepts of building on Aleo. You will be guided through the process of deploying and interacting with your own Aleo program directly on the web via the [Leo Playground](<https://play.leo-lang.org/>).

## 学习内容

  1. **Aleo 账户：** 学习如何创建 Aleo 账户并创建隐私和公开数据。
  2. **Aleo 程序：** 学习如何通过部署和执行您自己的程序与 Aleo 网络进行交互。
  3. **隐私与公开状态：** 学习如何在 Aleo 上管理隐私和公开数据，以创建隐私保护应用。


## 1\. 生成新的 Aleo 账户

在 Aleo 上构建的第一步是创建一个 Aleo 账户。Aleo 账户是一个唯一标识符，允许您 与 Aleo 网络交互。通过它您可以执行程序函数、创建自己的隐私和公开数据、 并部署新程序。您可以使用 Leo Playground 或生态钱包来创建账户。

### 1.1 使用 Leo Playground 小部件

点击账户小部件以生成新的 Aleo 账户。

![](../../../assets/images/new_account-964f37d64c1ca5ecf26bf2d9086bf530.png)


::: note[注意]
请将您的 **地址** 、**查看密钥** 和 **私钥** 保存在安全的地方，稍后会用到。
:::

### 1.2 使用钱包

可以通过以下方式创建账户： [Aleo Shield](<https://aleo.org/shield/>). 安装 Aleo Shield 后，系统将 提示您创建 Aleo 账户。

## 2\. 使用 Testnet 代币充值钱包

### 2.1 从水龙头获取代币

要充值钱包，您可以从官方水龙头请求代币： <https://faucet.aleo.org/>.

从水龙头收到的代币将是公开代币，这意味着它们在 Aleo 网络上公开可见。

### 2.2 验证 Testnet 代币的接收

在从上述水龙头之一收到成功确认后，您可以通过选择 `Get Account Balance` 从查询小部件的选项列表中，来检查您的账户余额。只需粘贴您的钱包地址并按 `Get`. 如果它们没有立即显示，别担心，您的 Testnet 代币可能需要几分钟才能到达。

![](../../../assets/images/account_balance-e3e3adad62afaa252944ff1653714fe8.png)

当代币到达时，按下以下按钮后，您应该在查询小部件中看到以下输出： `Get`.

![](../../../assets/images/provable_explorer-9e8c8589a342876a12002ed0cd561943.png)

您也可以在生态系统的任一区块浏览器上搜索水龙头提供的交易 ID。

  * [Provable Explorer](<https://testnet.explorer.provable.com/>)
  * [Aleoscan](<https://testnet.aleoscan.io/>)



::: tip[提示]
在等待代币发送到您的钱包时，您可以继续下一步。
:::

## 3\. 部署您的第一个 Aleo 程序

### 3.1 Token 程序

在本节中，您将学习如何通过 Leo Playground 将示例 `Token` 程序部署到 Aleo Testnet。 通过本节的学习，您将熟悉如何将自己的隐私保护程序部署到 Aleo。


::: info[信息]
`Token` 程序提供了一个简单的示例，说明如何通过创建 一个允许用户持有公开和隐私余额的代币来构建具有公开和隐私功能的程序。该程序将向您展示如何使用加密数据结构 （称为记录）来表示隐私数据，以及使用称为映射的公开链上数据存储来表示公开数据。
:::

### 3.2 自定义程序名称

要开始部署，请从 Leo Playground 的示例下拉菜单中选择 `Token` 程序。

![](../../../assets/images/select_token-86577b78405e146892dec88af167b104.png)

接下来，在 `src/main.leo` 的第 1 行自定义程序名称以使用唯一标识符，然后更新 `program.json` 文件中的 `program` 字段以匹配新名称。

![](../../../assets/images/program_name_-e42dd2005b28d6e8147c22c1683b8443.png) ![](../../../assets/images/program_json_-14eabc5b9ba123049267e8756be56e43.png)


::: tip[提示]
应用名称的长度应至少为 10 个字符。名称少于 10 个字符的部署费用更高。
:::

您现在可以部署第一个 Aleo 程序到 Testnet 了！在继续之前，请确保您的账户 已充值足够的 Testnet 代币以便部署到网络。

### 3.3 将程序部署到 Testnet

点击部署小部件以弹出部署窗口。在 `Program ID` 字段中输入您的程序名称，然后 输入您的私钥。您可以通过点击 `Estimate Fee` 来查看部署费用的估算值。准备好部署后，点击 `Deploy`。

![](../../../assets/images/deploy-3ef5e26d89c5df149205a9a23e97f200.png)

当您点击 `Deploy` 时，Leo Playground 将构建一个`部署交易`并提交到 Aleo 网络。 此过程可能需要一段时间。完成后，您应该会在弹出窗口中看到您的 Aleo 应用已部署的确认信息 其中包含交易 ID。

![](../../../assets/images/deployment-b4416bf9103633f7167d5f4519baecf9.png)

复制交易 ID 以在任何 Aleo 区块浏览器上查看部署交易的详细信息：

  * [Provable Explorer](<https://testnet.explorer.provable.com/>)
  * [Aleoscan](<https://testnet.aleoscan.io/>)



::: info[信息]
根据程序的大小，如果您的账户只使用了一次水龙头交易充值，可能没有足够的代币来部署程序。如果出现这种情况，您仍然可以与已部署到 Testnet 的程序进行交互。
:::

## 4\. 执行公开和隐私程序函数

您现在可以通过执行函数来使用已部署的 Token 程序了！


::: tip[提示]
如果您没有足够的资金来部署程序，您可以使用 [`token_quickstart.aleo`](<https://play.leo-lang.org/?gistId=b6730338a24169308348d5e38243665d&revision=3339199a4ac60976dc5ce6c0c35c5eefb0488ee0>) 程序来测试 Token 程序方法的执行。
:::

### 4.1 构建执行交易

当您调用程序的某个函数时，逻辑会在本地执行，同时生成一个零知识证明。该证明能够证实程序执行的正确性及其对应的输出，同时保持隐藏的输入或输出处于加密状态。函数执行完毕后，一个执行交易（Execution Transaction）会被提交到 Aleo 网络，其中包含执行摘要和正确性证明。当网络验证者验证了证明并确认支付了适当的费用后，该交易就会被接受。

### 4.2 隐私铸造代币

下面我们将使用 `mint_private` 函数隐私铸造一个代币。该函数将创建一个 Aleo `记录`，用于表示新的代币。记录是存储在链上的加密数据结构（以密文形式存在），只能由记录的所有者（使用所有者的查看密钥）解密。这确保了只有记录的所有者才能知道其内容，从而保持代币的私密性。

要隐私铸造代币，请导航到 Leo Playground 中的 Execute 小部件，并在 Program ID 选项卡中输入您的程序名称。或者，您也可以使用已部署的代币程序 `token_quickstart.aleo`。


::: tip[提示]
点击放大镜图标以确认 Program ID 对应于已部署的程序。
:::

![](../../../assets/images/execute-27d09e16a44353f45767bc285cf6b087.png)

接下来，输入您的私钥并从 Function 下拉菜单中选择 `mint_private` 方法。在 Inputs 选项卡中，输入您的账户地址和一个 `u64` 整数。


    ["ALEO_ADDRESS_HERE", "50u64"]

点击 Estimate Fee，然后点击 Execute。与之前的部署交易一样，您可以在区块浏览器中复制并粘贴您的交易 ID。您应该看到类似于以下内容：

![](../../../assets/images/mint_private-4ae194cf615e3a123ac39ce62afd21c6.png)


::: info[信息]
交易的输出是一个记录。要解密该记录，您需要复制并粘贴密文，导航到 Leo Playground 中的 Records 小部件，输入记录密文以及与您账户私钥对应的查看密钥，然后点击 Decrypt。解密后的记录应类似于以下内容：


    {
      owner: aleo1kypwp5m7qtk9mwazgcpg0tq8aal23mnrvwfvug65qgcg9xvsrqgspyjm6n.private,
      amount: 50u64.private,
      _nonce: 3175255370513411091535466147458245312227668453916963245036391157478647265587group.public
    }

要了解有关记录的更多信息，请点击 [here](../../../concepts/fundamentals/records/index.mdx)
:::

### 4.3 公开铸造代币

在 Token 程序中，公开余额存储在程序映射中。映射是 Aleo 网络节点上的公开键值存储，用于在链上存储长期的持久公开状态。

下面我们将使用 `mint_public` 函数铸造公开代币。当此函数运行时，它将更新与指定 Aleo `地址` 相对应的 `account` 映射中的余额。

返回到 Execute 小部件，并重复上一节中铸造公开代币的步骤。区块浏览器中的交易摘要应类似于以下内容：

![](../../../assets/images/mint_public-0a9450ace7432066abf47718ff709827.png)

与 `mint_private` 函数不同，`mint_public` 函数创建的执行交易包含一个 `Future` 而不是 `记录`。`Future` 包含一组供验证者在链上执行的指令，这些指令可以读取或更新映射。如果 `mint_public` 执行交易有效，验证者将接受它，并运行 `mint_public` `Future` 中更新 `account` 映射的指令。

通过在查询小部件中选择 `Get Mapping Value By Key` 选项可以查询映射值。您需要在相应的字段中输入 Program ID、映射名称和映射键。对于 Token 程序，映射键是拥有公开代币的 Aleo 地址。

![](../../../assets/images/quickstart_mapping-5e1908aa7e836b0cff26ceeb2ae93fb7.png)


::: info[信息]
太棒了！您已成功部署了一个 Leo 程序并在 Testnet 上执行了交易 🎉
:::

要开始本地开发，请导航到 [安装指南](../installation/index.mdx) 以安装必要的工具，将您的 Leo 程序部署到本地。
